บทที่ 10: สถาปัตยกรรม OOP (Object-Oriented Programming)

ยินดีด้วยค่ะ คุณเดินทางมาถึงบทที่สำคัญที่สุดในสายงานโปรแกรมเมอร์มืออาชีพ OOP คือการมองทุกอย่างในโปรแกรมให้เป็น "วัตถุ" (Object) ที่มีทั้งข้อมูลและพฤติกรรมในตัวเองค่ะ

10.1 Class vs Object

นึกภาพง่ายๆ ว่า Class คือ "แม่พิมพ์ขนม" ส่วน Object คือ "ขนม" ที่ถูกกดออกมาจากพิมพ์นั้นๆ ค่ะ

Attributes (ข้อมูล)

เปรียบเสมือน ค่านามธรรม ของวัตถุ เช่น ชื่อ, รุ่น, สี หรือปริมาณพลังงานที่มีอยู่

Methods (ความสามารถ)

เปรียบเสมือน การกระทำ ของวัตถุ เช่น การเดิน, การชาร์จไฟ หรือการทักทาย

10.2 หัวใจสำคัญของ OOP (The 4 Pillars)

การเขียนโปรแกรมเชิงวัตถุมีเสาหลักที่ทำให้โค้ดยืดหยุ่นและนำกลับมาใช้ใหม่ได้ง่าย:

  • Encapsulation: การเก็บข้อมูลไว้ในแคปซูล ป้องกันการแก้ไขมั่วๆ จากภายนอก
  • Inheritance: การสืบทอด (เช่น หุ่นยนต์รบ สืบทอดความสามารถมาจาก หุ่นยนต์พื้นฐาน)
  • Polymorphism: การที่หุ่นยนต์ต่างรุ่นกัน มีคำสั่ง "ทำงาน" เหมือนกันแต่ทำไม่เหมือนกัน
  • Abstraction: การซ่อนรายละเอียดที่ยุ่งยากไว้ข้างหลัง ให้เหลือแค่ปุ่มกดที่เข้าใจง่าย

10.3 การสร้าง Constructor (`__init__`)
