บทที่ 10: สถาปัตยกรรม OOP (Object-Oriented Programming)

ยินดีด้วยค่ะ Neet! คุณเดินทางมาถึงบทที่สำคัญที่สุดในสายงานโปรแกรมเมอร์มืออาชีพ OOP คือการมองทุกอย่างในโปรแกรมให้เป็น "วัตถุ" ที่มีทั้งคุณสมบัติและพฤติกรรมค่ะ

10.1 Class vs Object

ลองนึกภาพว่า Class คือ "แบบแปลนพิมพ์เขียว" ส่วน Object คือ "สิ่งที่สร้างขึ้นจากแปลนนั้น" ค่ะ

Attributes

คือ "ข้อมูล" หรือ "สถานะ" ของวัตถุ เช่น สีของรถ, ชื่อของพนักงาน, หรือเลือดของตัวละครในเกม

Methods

คือ "พฤติกรรม" หรือ "สิ่งที่ทำได้" เช่น รถสามารถเร่งความเร็วได้, พนักงานสามารถคำนวณเงินเดือนได้

10.2 หัวใจสำคัญของ OOP (The Pillars)

  • Encapsulation: การห่อหุ้มข้อมูลไว้ภายใน (ป้องกันการเข้าถึงจากภายนอกโดยตรง)
  • Inheritance: การสืบทอดคุณสมบัติ (เช่น คลาสหมา สืบทอดมาจากคลาสสัตว์)
  • Polymorphism: การที่วัตถุต่างชนิดกัน มีชื่อฟังก์ชันเหมือนกันแต่ทำงานต่างกัน
