บทที่ 9: การจัดการข้อผิดพลาด (Exception Handling)

เคยเขียนโปรแกรมแล้วมันหยุดทำงาน (Crash) ทันทีที่ผู้ใช้พิมพ์ข้อมูลผิดไหมคะ? นั่นแหละค่ะคือสิ่งที่เราจะแก้กันในบทนี้ เราจะเรียนรู้วิธี "ดักจับ" ข้อผิดพลาดเพื่อให้โปรแกรมยังรันต่อไปได้สวยๆ ค่ะ

9.1 รู้จักกับ Common Exceptions

ก่อนจะแก้ เราต้องรู้ก่อนว่า Python มี Error ยอดฮิตอะไรบ้าง:

  • ZeroDivisionError: เมื่อพยายามหารตัวเลขด้วยศูนย์
  • ValueError: เมื่อส่งค่าผิดประเภท (เช่น เอาคำว่า "สิบ" ไปแปลงเป็นตัวเลข)
  • FileNotFoundError: เมื่อพยายามเปิดไฟล์ที่ไม่มีอยู่จริง
  • TypeError: เมื่อใช้ Operator ผิดประเภทข้อมูล

9.2 โครงสร้าง Try-Except

นี่คือหัวใจหลักค่ะ เราจะเอาโค้ดที่ "เสี่ยง" ไปวางใน try และวางแผนรับมือใน except

Syntax:
try: (ลองทำอันนี้ดูนะ)
except ErrorName: (ถ้าเจอปัญหาชื่อนี้ ให้แก้แบบนี้นะ)
else: (ถ้าไม่มีปัญหาเลย ให้ทำอันนี้นะ)
finally: (ไม่ว่าจะเกิดปัญหาหรือไม่ ต้องทำอันนี้เสมอ เช่น ปิดไฟล์)
