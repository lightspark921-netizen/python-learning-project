บทที่ 8: การจัดการไฟล์และระบบข้อมูล (File I/O)

ความแตกต่างระหว่างโปรแกรมธรรมดากับซอฟต์แวร์ระดับมืออาชีพ คือความสามารถในการจัดเก็บข้อมูล (Persistence) บทนี้จะสอนการอ่านและเขียนไฟล์เพื่อบันทึกสถานะของโปรแกรมลงในระบบ

8.1 File Access Modes

การเปิดไฟล์ใน Python ต้องระบุโหมดให้ชัดเจน เพื่อป้องกันการเขียนทับโดยไม่ตั้งใจค่ะ:

Modeคำอธิบาย
'r'Read: อ่านอย่างเดียว (Default) - จะ Error ถ้าไม่มีไฟล์
'w'Write: เขียนทับ - จะสร้างไฟล์ใหม่ถ้ายังไม่มี แต่ถ้ามีจะลบของเก่าทิ้ง
'a'Append: เขียนต่อท้าย - ข้อมูลเก่าจะไม่หายไป

8.2 The Best Practice: Context Manager

การใช้ with open(...) as f: เป็นวิธีมาตรฐานที่ปลอดภัยที่สุด เพราะ Python จะปิดไฟล์ให้เราอัตโนมัติเมื่อทำงานเสร็จ ป้องกันปัญหาไฟล์เสีย (Corrupt)

Syntax ที่แนะนำ:
with open('memo.txt', 'w') as file:
    file.write("บันทึกข้อมูลสำเร็จ!")
